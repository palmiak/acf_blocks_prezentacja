14Tworzymy blok

                                    add_action('acf/init', 'my_acf_init');
function my_acf_init() {
    if( function_exists('acf_register_block') ) {
        acf_register_block([
            'name'  => 'trips', // nazwa
            'title' => __('Trips'), // wyświetlany tytuł
            'description' => __('Promoted trips'), // opis
            'render_callback' => 'my_acf_block_render_callback', // nazwa funkcji renderującej
            'category' => 'layout', // kategoria
            'icon' => 'location-alt', // ikona
            'keywords' => [ 'trips', 'flights' ], // słowa kluczowe
        ]);
    }
}
Następny slajd - więcej opcji
- opisać wszystkie pola
- keywords - do wyszukiwania bloku po słowue kluczowym
- jako ciekawostka - na obecnie chwilę są problemy z atrybutem icon - nie można użyć inline svg

16Tworzymy blok

                                    function my_acf_block_render_callback( $block, $content='', $is_preview ) {
    $context = Timber::get_context();
    // Wszystkie informacje o bloku
    $context['block'] = $block;
    // Wszystkie pola użyte w bloku
    $context['fields'] = get_fields(); 
    // Sprawdzamy czy to podgląd w panelu admina.
    $context['is_preview'] = $is_preview;
    // Renderujemy blok.
    Timber::render( 'views/blocks/trip-block.twig', $context );
}
Następny slajd - ustawienia
- rozdzielam logikę od wyglądu - Timber wręcz wymusza takie zachowanie
- block - wszystkie informacje o bloku jak np ID, class
- fields - wszystkie pola z acf
- is_preview - jeżeli true oznacza że jesteśmy w panelu admina, jeżeli false to front strony
- render - podpięcie templatki - nie będę pokazywał kodu, bo jest go stosunkowo dużo i raczej nam się tu nie przyda.

32Wnioski

  • Brak stabilności Gutenberga w połączeniu z wersją beta ACF Blocks powodują, że używanie ich na produkcji może napsuć nam trochę krwi
  • ACF Blocks ma ograniczenia jeśli chodzi o dodatkowe elementy interface'u (np. sidebar czy dodatkowe menu kontekstowe) - trzeba więc bardzo poważnie pomyśleć jak stworzyć wygodny interface
  • ACF Blocks nie współpracuje na obecną chwilę z "reusable blocks"
Następny slajd - moja fotka
- Stabilność może być sporą przeszkodą - ale są sytuacje kiedy warto popróbować - mniejsze projekty, bardziej zaufany klient etc.
- UI to największe ograniczenie - dzięki reactowi można użyć fajnych elementów - od różnych elementów w sidebarze, po kontektowe menu etc - ACF daje nam formularze, taby, akordeon
- na początku był problem we współpracy z kolumnami (ale to już działa) - nie działa za to reusable blocks - dla niektórych może być to problem - ja akurat uznaję, że bloki są samodzielnymi bytami a nie sklejką z innych - nie ufam userom
